Timetracker 0.1 XRX Demo application

This application uses the XRX (XForms Rest XQuery) web application architecture. Essentially this means that all functionality was built using XML technologies from front to back. To learn more about XRX please refer to the XRX Wikibook

Timetracker is a simple application to track the working time spend on different projects. It's an example of a CRUDS (create, read, update, delete and search) application but not only stores data into the database but also keeps the whole application along with all its XForms, XQuery and resource files in a database collection.

Several aspects of a XRX application are demonstrated by Timetracker:

  • it uses XForms for data entry and validation
  • XForms documents are dynamically generated by XQueries
  • data are accessed via the REST interfaces provided by the database
  • XQuery is used for searching and filtering the datasets

Besides that certain betterFORM-specific capabilities are demonstrated:

  • betterFORM extends the XForms 1.1 specification by providing a submission mode of 'embedHTML'. This means that query params are gathered in a normal XForms instance, submitted with GET to a XQuery in the database and the resulting plain HTML is embedded at a defined target in the page that send the request. This avoids to store the returned data in a XForms instance and the rather costly presentation in a XForms repeat. The result table is implemented using this feature.
  • Similar to this Timetracker uses an extension to the XForms 'load' action to allow authors to embed document fragments identified by an id into the current page. This works for plain HTML documents but also for XForms models. The 'create' and 'edit' features use this to embed a XForms into a dialog window for editing.

Timetracker was developed by the betterFORM team on top of eXist 1.4 and betterFORM limegreen XRX edition.
